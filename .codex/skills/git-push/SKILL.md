# Git Push

## Overview

Push commits safely by confirming the current branch and remote target before pushing.

## Workflow

1. Check current branch and summarize ahead/behind status.
2. Confirm the remote/branch target if unclear.
3. Push to the confirmed target.

## Notes

Push only after confirming the user wants to publish the current branch.
