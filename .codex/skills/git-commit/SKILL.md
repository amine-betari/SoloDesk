# Git Commit

## Overview

Create consistent, safe commits in this repository by following a short, explicit workflow before committing.

## Workflow

1. Check the working tree status and summarize what changed.
2. If changes are unclear or broad, ask for scope confirmation.
3. Propose a concise commit message using conventional prefixes (`feat:` or `fix:`).
4. Wait for user approval of the message.
5. Commit using the approved message.

## Notes

Use non-interactive git commands only.
